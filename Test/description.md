# BinaryVersionManager (二进制多版本管理器)

这是一个用于高效管理和存储二进制文件多版本的 Kotlin 类。它采用“基础版本 + 增量补丁 (Base + Delta Patch)”的思路，极大地减小了多版本文件的存储空间占用。

** Justinz.注：算法是AI写的，没有引入库，若需更高效还要更改 **

## 核心设计理念

传统的版本保存方式通常会直接复制一份完整的新文件，这在文件较大或是版本更新频繁时，会造成极大的存储空间浪费。
`BinaryVersionManager` 通过手写的 **二进制差异对比算法 (Diffing Algorithm)** 解决了这一问题：

1. **基础版本 (Base Version)**：首次创建时，会将完整的文件数据作为“基础版本”保存下来（以 `.bin` 结尾）。
2. **增量补丁 (Delta Patch)**：后续每一个新版本，只会与前一个版本进行二进制层面的对比，仅提取出被修改、增加或删除的部分数据流，生成极小的“补丁包”（以 `.patch` 结尾）。
3. **版本还原 (Reconstruction)**：当需要读取某个历史版本时，程序会先加载基础版本，然后按顺序依次打上补丁，从而“推演”出并还原该版本原本的完整状态。

## 目录结构

自行设定文件夹以存放文件，目录结构大致如下：

```text
📁 ./Files/
 ├─ 📄 manifest.txt      (记录所有已保存版本的先后顺序清单)
 ├─ 📄 v1.0.bin          (第一个核心全量基础版本)
 ├─ 📄 v1.1.patch        (v1.1版本相对于v1.0版本的增量改动)
 ├─ 📄 v1.2.patch        (v1.2版本相对于v1.1版本的增量改动)
 └─ 📄 v2.0.patch        ...依此类推
```

## 核心功能接口 (API)

### 1. `availableVersions: List<String>`
* **功能**：获取当前归档中所有有效、可用的版本号列表。
* **机制**：直接读取并解析 `manifest.txt` 中的记录。

### 2. `createNewArchive(initialData: ByteArray, versionId: String)`
* **功能**：从零开始创建一个全新的多版本归档库。
* **使用场景**：首次导入文件，或需要完全重置版本库时调用。
* **机制**：清空现有的 `./Files/` 目录的内容，将传入的 `initialData` 保存为首个全量 `.bin` 文件，并在 `manifest.txt` 中写入该版本号。

### 3. `exportNewVersion(newData: ByteArray, newVersionId: String)`
* **功能**：导出一个新的版本。
* **使用场景**：当文件发生了修改，需要增加一个新版本记录时调用。
* **机制**：
  - 自动提取当前最新的版本数据。
  - 调用底层的 `computeDelta` 算法，计算出 `newData` （新改动的数据） 与旧数据之间的差异。
  - 将生成的补丁包写入到以 `.patch` 为后缀的文件中。
  - 更新记录清单。

### 4. `importVersion(targetVersionId: String): ByteArray`
* **功能**：读取并还原出您指定的某个具体版本的内容。
* **使用场景**：用户请求导出旧版本、回退版本、或单纯想要读取当前最新状态时调用。
* **机制**：读取 `.bin` 基础全量文件，并在内存中循环遍历，依次应用（`applyDelta`）从基础版本到目标版本之间所有的 `.patch` 增量包，最终返回拼接完成的完整二进制 `ByteArray`。

## 底部算法解析 (手写区块)

为了避免引入庞大的第三方库，类内部手写了针对二进制块对比的微型增量算法：

* **`computeDelta(...)`（差异提取器）**：
  采用哈希步长机制为旧数据构建字典索引，并设置了 `minMatchSize = 8` 作为最小防碎片匹配块。读取新数据时，若能在旧字典中找到匹配区块，则向补丁文件写入一根**指令 0 (复用 Copy)**，让它直接从旧数据的某个偏移量“借用”长度数据；若发现全新的异化数据，则写入一根**指令 1 (新增 Add)**，将这段新数据实体嵌入补丁内保留。
* **`applyDelta(...)` (补丁应用器)**：
  流式解析生成的补丁内部结构。遇到复用指令，则向缓冲区填充上次老旧文件的旧区段；遇到新增指令，则向缓冲区注入补丁自带的新区段；遇到**指令 2 (EOF)** 则宣告全序列拼接完成。
